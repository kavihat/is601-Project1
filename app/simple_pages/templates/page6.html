Testing

Testing is a procedure that exercises a behavior to see if the programs which are tested function according to the clients' specifications. Testing can be done manually or be automated. The automated method is faster and accurate. They are more suitable for scenarios where test cases are run repeatedly for a longer duration of time. Manual testing helps to find user-friendliness of a feature, as in the cases of Web UI testing.

Unit Testing and its Advantages

Unit testing involves testing the individual components of the application or software program. It tests the smallest possible component that is publicly exposed.

  • Process becomes agile as adding new changes to old code is less expensive.
  • It helps the developer identify the smallest deviations from expected behavior, and improves code quality to a great extent.
  • The debugging process is simplified to a great extent by unit testing. If any test fails, only the latest changes have to be debugged.

What is AAA?

AAA is a pattern for structuring tests. It divided the testing method into three stages: Arrange, Act, Assert. Each stage leads to the next i.e., the output from the first stage is the input to the second stage and so on. Arrange is called first; the output of Arrange is used to call Act and finally, the result of Act is tested using Assert.

  • Arrange. Organizes inputs and targets. It could involve initializing object settings, prepping a database, logging into web applications etc. It handles all the operations at start of the test
  • Act. This is when the system is under test. This could involve calling the method or API being tested and capturing its output/effect.
  • Assert. This allows you to make claims about the output recorded in the Act step. Matching the output against the expected result verifies the correctness of the response. Thus, the test passes or fails based on the return value, final state etc.
Sometimes there may be a series of Arrange, Act or Assert operations appearing in a test. It's always better to break it down to individual unit tests as that is what makes these tests powerful.

AAA Testing with Python

Consider the following Python code that illustrates the AAA testing code. This code snippet shows the Arrange, Act and Assert steps in the test case for checking the addition operation of a Calculator object. The same test case can be done without the AAA pattern, but using AAA makes it structured and readable.

Using AAA pattern
def test_diplayed_name():
   """
   Test case to show the AAA pattern
   """
   # Arrange
   calc1 = Calculator()
   # Act
   output = calc1.add(3,4)
   #Assert
   assert output == 7

Here, we instantiate the Calculator object at the Assert stage. Once an object is instantiated, the add method of the Calculator class is called and the output of that operation is stored in a variable. The Assert stage compares the returned value with the expected value.

Benefits of AAA testing

  • AAA pattern helps to structure tests.
  • Each step of the pattern has its job to do, and this separation of what is being tested from the setup and result verification makes the code easier to read and understand.
  • Simplicity and readability enhances the maintainability, review and debugging of the tests.
