Object Oriented Programming

Object Oriented Programming is a programming paradigm that relies on the concept of classes and objects. It helps to structure the software program into simple, reusable pieces of code blueprints which are used to create individual instances of objects.

Object Oriented Programming is important because it makes visualization easier by being close to real-world scenarios and helps in implementing real world like entities.

Popular object-oriented languages include Java, C++, Python, and C#. The primary philosophy of OOP is to bind the data and the functions that work on that together as a single unit so that no other part of the code can access this data.

The main OOP concepts are:

  • Data Abstraction
  • Encapsulation
  • Inheritance
  • Polymorphism

Data Abstraction

Abstraction is hiding the internal details or implementations of a function or object and only exposing its external behavior. In Python, any function abstracts the inner implementation from its caller and exposes only its behavior via return values and side effects. Further, abstract classes can be used to model real-world behavior by enumerating functionality expected from any non-abstract class that is based on the abstract class. In the following example, abc.ABC is used to create an abstract base class (ABC) which defines the behavior expected from its child classes. 

from abc import ABC, abstractmethod

class Operation(ABC):
   """class operation for specifying operations"""
   @abstractmethod
   def get_result(self):
       """Creating an interface which cannot be modified but extended as needed"""
       pass

Here we are creating an Operation class as the abstract class. It declares an abstract get_result() method, which is the interface which will be implemented in any class that inherits the Operation class. The child class needs to provide the definition and these definitions are hidden from the user.

Encapsulation

Encapsulation describes the idea of wrapping data and methods that work on data as a unit. This enhances security by restricting accessing variables and methods directly and prevents accidental modification of data. This is achieved by using private variables for an object that can be changed only with an object method. 

class Multiply(Operation):
   """performing the multiplication"""
   def __init__(self, value1, value2):
       self.__value1 = value1
       self.__value2 = value2

   def get_result(self):
       return self.__value1 * self.__value2

To define a private member in Python, prefix the member name with double underscore. As you can see in the code snippet, __value1 is a private member that cannot be accessed by users but can be modified by class methods. It is the idea of wrapping the variables (__value1, __value2), with the methods that act on them (get_result()) as a unit that is described as Encapsulation.

In reality, there are no “private” instance variables in Python that cannot be accessed outside an object. However, a convention being followed by most Python developers is to treat __variables as a non-public part of the API.

Polymorphism

Polymorphism is a Greek word where poly means many and morph means forms. In OOP, it refers to the functions that have the same names but carry different functionalities.

Consider the below code snippet, here we have + operator that adds 2 operands and returns an integer value if the operands are integers. The same operator also returns a float value if the operands are floats and returns a concatenated string if the operands are strings. The behavior changes based on the arguments, if the function name remains the same.

class Add(Operation):
   """performing the addition"""
   def __init__(self, value1, value2):
       self.__value1 = value1
       self.__value2 = value2

   def get_result(self):
       return self.__value1 + self.__value2

When + operator is used, it assigns the arguments to variables and performs “+” operation on them regardless of the data type being integer, float or string. This behavior is called Polymorphism

Inheritance

Inheritance is the mechanism in which one class inherits attributes and methods from another class. The class being inherited from is called parent class or base class.The class that inherits from another class is called child class or derived class. 

from abc import ABC, abstractmethod

class Operation(ABC):
   """class operation for specifying operations"""
   @abstractmethod
   def get_result(self):
       """Creating an interface which cannot be modified but extended as needed"""
       pass
            

class Add(Operation):
   """performing the addition"""
   def __init__(self, value1, value2):
       self.__value1 = value1
       self.__value2 = value2

   def get_result(self):
       return self.__value1 + self.__value2

Here, Operation is the base class and Add is the subclass. Add inherits the declaration of the method get_result() from the base class for which it provides its own implementation (adding operands and returning the sum).
