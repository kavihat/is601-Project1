Abstraction

The process of hiding details or attributes in an object or system to highlight details of greater importance, thereby reducing complexity.

Abstract Method

A method that has a declaration but does not have an implementation. Defines the signature of a class method that can be concretely defined in a subclass. Implemented in Python using the @abstractmethod method decorator.

Abstract Class

A class that cannot be instantiated, but can be subclassed. An abstract class may or may not include abstract methods, but any concrete subclass must implement abstract methods inherited from the abstract base class. An abstract base class (ABC) provides a common interface for any concrete implementation of a component. Implemented in Python by having the abstract class derive from abc.ABC.

Class

A blueprint to define an entity or system as a collection of related behaviors and internal state. A class is used to create individual instances of these entities (objects), providing initial values for state (member variables) and implementations of behavior (member functions or methods).

Constructor

A special method within a class that is used to create and return an object of that class, and the created object is called an instance of the class. The constructor set the initial state of the created objects. In many languages, the constructor method has the same name as the class itself. In Python, the constructor method __new__() creates a new object, while the __init__() method sets initial values for created objects.

Design Patterns

General reusable solution to a commonly occurring problem in software design. Design patterns prescribe a template to solve a problem that is applicable in many situations, without providing a concrete design. Many software developers use design patterns as a formal description of best practices that are used in software design. Examples include the Factory Method, Builder, Decorator, Observer, and Visitor design patterns.

Encapsulation

In the context of Object-Oriented Programming, encapsulation refers to the concept of bundling data and the methods that operate on the data, into a single unit. Encapsulation is often provided by classes, which hides the state of the structured data inside an object, instead enforcing controlled access to these state variables and implementation details by the use of getter and setter methods.

Fixture

Function that is attached to a test function and runs before the test to which it is attached. Fixtures are used to feed data to a test, such as database connections, file handles, or input data. Fixtures allow multiple tests to share an initial environment without having to repeat code. In Python, using the pytest library, the @pytest.fixture annotation marks a function as a fixture that can be attached to unit tests or other fixtures.

Inheritance

In the context of Object-Oriented Programming, inheritance refers to the mechanism by which a class (called a child or derived class) is built upon, and acquires properties — such as data members and member functions — of another class (called parent or base class) or classes. Inheritance facilitates code reusability, and allows a class to derive behaviors from its parent class(es) without having to duplicate the implementation. Inheritance can also allow a class to provide alternate implementations for the same behavior, i.e., maintaining the same interface.

Instance Method

Instance methods are functions defined in a class that are executed within the context of a created class object (instance). By running within an instantiated object, and possessing a reference to the instance to which it is bound (called self in Python), instance methods are able to access instance variables, class and static variables, and call other instance methods as well as class and static methods.

Instance property

Instance property (also, instance attribute) is variable defined within a class that is bound to (or belongs to) only one object instance of the class. In other words, each instance of a class possesses its own set of instance properties. Most commonly instance properties are declared and initialized within the class constructor.

Instantiation

Instantiation in Object-Oriented Programming refers to the creation of an instance (object) of a class, most commonly through the use of the constructor method. Instantiation also sets initial/default values for instance attributes. In Python, instantiation using the constructor takes the form: 

object_name = ClassName(args…)

Namespace

A namespace is a named grouping of objects, functions and types, created during the execution of a Python program. Every variable declaration, function definition, module and package is placed within a namespace within which they are resolved. Namespaces are used to resolve names unambiguously, and to define the scope and lifetimes of the names defined within them.

Object

An object is the product of a class instantiation (instance of a class). It is an allocation in memory of instance properties, bundled with the methods that act on these properties.

Polymorphism

Mechanism in Object-Oriented Programming where an object or function can take on different forms based on the context in which it is called or accessed. E.g., the + operator has different polymorphic forms and behaves differently depending on the operands that it is given — integers, floats, strings or lists.

Type Cast

An implicit or explicit conversion of an object to a different type. Type casting can be used explicitly to enforce specific behavior within a polymorphic function, reframe an object of a child class as an object of a parent class, or implicitly to prevent data loss.

Type hint

Type hints is a feature of Python (and other dynamically typed languages) that allows a programmer to explicitly declare the data-type of a variable when declaring it. This increases readability, allows the interpreter to sanity-check data types during runtime, and allows the programmer to signal which behavior of a polymorphic function or object is expected.

SOLID

SOLID is a group of five design principles used in Object-Oriented Programming — Single Responsibility Principle, Open/Closed Principle, Liskov Substitution Principle, Interface Segregation Principle, and Dependency Inversion Principle. The aim of the SOLID principles is to increase code readability, and build maintainable and extensible code.

Static

Static is a property modifier that an entity is related to an object or class, but is not bound to any specific instance of that class. Static objects have a lifetime throughout the lifetime of the program (even when not in scope).

Static Method

Static methods are functions that are created within a class namespace that relate to the behavior of a class, but do not control or access the internal state of the class or any class object. These are utility functions that express the behavior of a class, but do not reference the class or class object.

Static property

Static properties are variables that are created within a class namespace, but do not directly represent the internal state of a class object. They are declared outside any class method, and do not change between class instances.

Unit Testing

A unit test is a short program fragment which exercises some narrow part of the product’s source code and checks the results. The result of a unit test is binary — either a pass or a fail. Unit tests check an individual module in isolation, and therefore are not typically dependent on external systems, services or applications. Pytest is the most popular unit testing framework for Python.
